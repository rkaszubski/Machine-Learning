DATASET: https://archive.ics.uci.edu/ml/datasets/seismic-bumps

Problem: Predicting if a seismic hazard will occur while mining

The Dataset has 2584 observations and 18 variables (excluding the class variable). The variables include energy levels, pulses, seimsic bumps, among a few others. The target variable is refered to as "class" with a 0 representing a nonhazard and 1 representing a hazard.

Methodology:

The goal is to create several classifiers that will best predict whether or not a seismic hazard will occur. To start I'm going to load in the dataset and do some data exploration. This will include looking at the different variables, their distributions, their coorelations, and creating a few different visualization to better represent this.

After this, I will prepare the data for machine learning by converting any categoricaly variables to dummies and splitting the dataset into training and testing. From there I will use several different machine learning models included in the sklearn package, most of these were covered in class and used in previous assignments, but a few I decided to try on my own. These will be evaluated looking at the different metrics. My primary focus is to find models that excel at predicting hazards (1's). There is a large class imbalance with only 6% or so of the dataset being 1's. Therefore it is key to find a model with a high balanced accuracy, f1 score, and recall.

Note: I reran this entire notebook before submitting, so there may be some differences in the results for certain models than what I wrote in the comments

The objective here is to be able to predict those hazards. It is far more important for a model to predict hazards than to predict non-hazards. Predicting that something is hazardous will at least allow the miners to take the proper precautions. If you predict something as non-hazardous but it ends up being hazardous, then you might have devastating consequences because the proper precautions won't be taken. That's why I'm not as concerned about a very predicting accuracy for 0's. Of course, the best model should be able to predict both relatively well, but the key is to be able to predict the hazards.

This model is able to do that. Yes the percision is a little low because there is many false positives. But it's not that much lower than the second best model. And again, in a real world scenario, a false positive is not ideal but it's okay and a better outcome than predicting a false negative.

Out of all the models, this one has the highest recall at .65, it has the highest balanced accuracy at .74, it is tied for the highest f1 score with .32. It is definetly the best and most useful model of the ones attempted.

When it comes to predicting sort of natural hazards or disasters like seimsmic activity, there is a lot more data of cases were it isn't hazardous than hazardous. So, in this dataset I only had 2500 data points to begin with and of that only 6% or so were actually hazardous. This means that there isn't a lot of data for the models to be trained out, and also when evaluating we get misleading accuracies because it's predicting the majority class very well and not the small minority class.
